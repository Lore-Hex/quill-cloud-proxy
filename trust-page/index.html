The Quill Cloud prompt path runs inside an attested workload that terminates TLS internally. The current production endpoint is a GCP Confidential Space workload; the AWS Nitro Enclave path remains documented below for the Bedrock deployment. In both cases, the only code that decrypts your prompt is the workload whose published measurement is checked against live attestation evidence.

Live attestation

The GCP workload returns a Google-signed Confidential Space JWT. The device verifies it before sending prompt traffic: JWT signature, audience, issuer, caller nonce, live TLS cert fingerprint in eat_nonce, and submods.container.image_digest matching /image-digest-gcp.txt. 

NONCE="$(openssl rand -hex 16)"
curl -kSs "https://api.quillrouter.com/attestation?nonce=$NONCE" -o attestation.jwt

# The JWT issuer discovery document is:
# https://confidentialcomputing.googleapis.com/.well-known/openid-configuration
# Check:
#   1. RS256 signature chains to Google's Confidential Space JWKS
#   2. aud == quill-cloud and iss == https://confidentialcomputing.googleapis.com
#   3. eat_nonce contains $NONCE and the live TLS cert SHA-256
#   4. submods.container.image_digest matches /image-digest-gcp.txt

The AWS Nitro path returns a COSE_Sign1 CBOR document. For the AWS deployment, verify it with: 

curl -kSs https://api.quillrouter.com/attestation -o attestation.cbor

# Decode + verify with AWS Nitro's root CA, then check:
#   1. COSE_Sign1 chains to AWS Nitro Attestation PKI
#   2. PCR0 matches /pcr0.txt above
#   3. public_key field matches the cert your TLS handshake received
#   4. user_data[:32] matches the hash of that cert
uv run --with cbor2 --with cryptography python3 \
       tools/verify-attestation.py attestation.cbor \
       --expected-pcr0 "$(curl -sS https://trust.quill.lorehex.co/pcr0.txt)" \
       --api-host api.quillrouter.com

Each attestation document binds this exact build (GCP image digest or AWS PCR0) to the TLS cert presented in your handshake. If the cert rotates on a new workload boot, the device refetches attestation and pins the newly verified cert. The cert-private-key never touches disk and never leaves workload memory.

The check is the same shape Brave's nitriding-daemon uses; the bindings (PCR0/image digest → attestation document → cert public key/fingerprint → live TLS session) chain to "the bytes I'm sending are encrypted to a specific build of code measured against this repo."

Code transparency

Every release of the trust files is signed by the GitHub Actions OIDC identity for Lore-Hex/quill-cloud-proxy and recorded in Sigstore Rekor, a public append-only transparency log. The signature + cert + Rekor inclusion proof are published next to each file as .bundle. Anyone can verify the GCP digest binding without trusting this server: 

curl -sS -o image-digest-gcp.txt        https://trust.quill.lorehex.co/image-digest-gcp.txt
curl -sS -o image-digest-gcp.txt.bundle https://trust.quill.lorehex.co/image-digest-gcp.txt.bundle

cosign verify-blob \
  --bundle image-digest-gcp.txt.bundle \
  --certificate-identity-regexp '^https://github.com/Lore-Hex/quill-cloud-proxy/.*' \
  --certificate-oidc-issuer 'https://token.actions.githubusercontent.com' \
  image-digest-gcp.txt

A successful verify proves: this digest was signed by the GitHub Actions identity for this repo, at the timestamp Rekor recorded, against a Fulcio-issued ephemeral certificate whose subject claims matched the repo's CI run. A silent rebuild of the cloud binary can't forge this — Rekor is append-only and the OIDC token is short-lived.

What enforces zero retention

  • enclave-go/cmd/enclave/main.go — the workload prompt path, bearer authentication, request-size cap, and attestation endpoint.
  • parent/tests/test_no_content_in_logs.py — AST scan that fails CI if a parent log call has any prompt-content kwarg.
  • parent/src/quill_parent/relay.py — the byte pump between the ALB and the enclave; never decodes the body.
  • quill-cloud-infra/modules/kms/main.tf — KMS condition policy that binds Bedrock-credential decrypt to the published PCR0.
