The Quill Cloud prompt path runs inside an AWS Nitro Enclave that terminates TLS internally. AWS infrastructure (the NLB, the parent host) only ever sees ciphertext; the only code that decrypts your prompt is the binary measured by the PCR0 below. The KMS keys the enclave needs to reach Bedrock are released by AWS only when that PCR0 matches the published value, binding the running binary to this open-source code at this commit.

Live attestation

The enclave terminates TLS internally — AWS infrastructure (ALB, NLB, parent host) never holds plaintext prompt content. To verify a live session, fetch the attestation document from the running enclave: 

curl -kSs https://api.quill.lorehex.co/attestation -o attestation.cbor

# Decode + verify with AWS Nitro's root CA, then check:
#   1. COSE_Sign1 chains to AWS Nitro Attestation PKI
#   2. PCR0 matches /pcr0.txt above
#   3. public_key field matches the cert your TLS handshake received
#   4. user_data[:32] matches the hash of that cert
uv run --with cbor2 --with cryptography python3 \
       tools/verify-attestation.py attestation.cbor \
       --expected-pcr0 "$(curl -sS https://trust.quill.lorehex.co/pcr0.txt)" \
       --api-host api.quill.lorehex.co

The attestation document binds this exact build (PCR0) to the public key of the TLS cert presented in your handshake. If the cert rotates (new enclave boot), refetch /attestation and repin. The cert-private-key never touches disk and never leaves the enclave's memory.

The check is the same shape Brave's nitriding-daemon uses; all four bindings (PCR0 → attestation doc → cert public key → live TLS session) chain to "the bytes I'm sending are encrypted to a specific build of code measured against this repo."

Code transparency

Every release of pcr0.txt is signed by the GitHub Actions OIDC identity for Lore-Hex/quill-cloud-proxy and recorded in Sigstore Rekor, a public append-only transparency log. The signature + cert + Rekor inclusion proof are bundled at /pcr0.txt.bundle. Anyone can verify the binding without trusting this server: 

curl -sS -o pcr0.txt        https://trust.quill.lorehex.co/pcr0.txt
curl -sS -o pcr0.txt.bundle https://trust.quill.lorehex.co/pcr0.txt.bundle

cosign verify-blob \
  --bundle pcr0.txt.bundle \
  --certificate-identity-regexp '^https://github.com/Lore-Hex/quill-cloud-proxy/.*' \
  --certificate-oidc-issuer 'https://token.actions.githubusercontent.com' \
  pcr0.txt

A successful verify proves: this PCR0 was signed by the GitHub Actions identity for this repo, at the timestamp Rekor recorded, against a Fulcio-issued ephemeral certificate whose subject claims matched the repo's CI run. A silent rebuild of the cloud binary can't forge this — Rekor is append-only and the OIDC token is short-lived.

What enforces zero retention

  • enclave/tests/test_no_io.py — AST scan that fails CI if any I/O identifier (print, open, logging, etc.) appears in the enclave production source.
  • parent/tests/test_no_content_in_logs.py — AST scan that fails CI if a parent log call has any prompt-content kwarg.
  • parent/src/quill_parent/relay.py — the byte pump between the ALB and the enclave; never decodes the body.
  • quill-cloud-infra/modules/kms/main.tf — KMS condition policy that binds Bedrock-credential decrypt to the published PCR0.
